Apuntes de clase

Conglomerados

  • Goal: Agrupar observaciones, de modo que cierta distancia entre observaciones, por grupo, sea minimizada.

Intro

  • Clustering es una herramienta para anÃ¡isis descriptivo.

  • El objetivo principal es encontrar subgrupos homogÃ©neos.

  • Se busca que observaciones del mismo grupo sean similares; mientras que, de grupos distintos, sean diferentes.

  • Clustering no es clasificar, pues no hay variables dependiente.

  • Por lo general, no es adecuado comparar cluster algorithms diferentes.

  • Clustering tambiÃ©n puede servir para reducir el tamaÃ±o de un conjunto de datos.

Tipos de AnÃ¡lisis Cluster

  • Cluster jerÃ¡rquico
  • Cluster Bayesiano
  • Cluster particional

Problemas comunes

  • Problemas principales:

    • AglutinaciÃ³n: Ocurre cuando un objeto puede pertenecer a mÃ¡s de un cluster.
      Para mitigar esta situaciÃ³n, se usan algoritmos difusos.
    • DisecciÃ³n: Ocurre cuando la poblaciÃ³n asociada a los datos no contiene clusters significativos.

  • Presencia de outliers.

  • Datos no normalizados (no tener misma escala).

Similaridad, distancia y proximidad

  • Medida o funciÃ³n de similaridad:
    • Cuantifica similaridad entre objetos.
    • A mayor valor, objetos mÃ¡s similares.
    • Usualmente es un valor entre 0 (no similaridad) y 1 (objetos completamente similares.)
  • Distancia (medida de disimilaridad):
    • Indica quÃ© tan diferentes son los datos.
    • Menor valor, objeto smÃ¡s parecidos.
    • MÃ­nimo valor de disimilaridad suele ser 0.
    • Rango suele ser \(\left[0; 1\right]\) o \(\left[0; \infty \right(\)
  • Proximidad: Se refiere usualmente tanto a la similaridad como a la disimilaridad.

Matriz de distancias (disimilaridad)

  • Matriz \(n\times n\) donde se registran las distancias \(d \left( i, j \right)\) .
  • Usualmente simÃ©trica (no siempre).

Las funciones de distancias son usualmente distintas para variables de tipo cuantitativa o categÃ³rica.

Distancias con datos numÃ©ricos:

  • Distancias de Minkowski: SÃ­ es una mÃ©trica matemÃ¡tica. \[ d \left( i, j \right) = \left( \sum_{k=1}^{M} | x_{ik} - x_{jk} |^p \right)^{\dfrac{1}{p}}, \]

donde los datos pertenecen a \(\mathbb{R}^M\), y \(p\) es el orden de la distancia (\(L_p norm\) ).

  • Distancia de Canberra

Distancias con datos mixtos

  • Distancia de Gower (el profesor no recomienda usar esta distancia, dice que no tiene un sustento robusto, a menos que no tengamos una mejor opciÃ³n)

  • Como alternativa, el profesor recomienda usar un MD Cluster.

MÃ©todos de Particionamiento

  • Los datos son particionados en un nÃºmero \(K\) fijado de conglomerados.
  • Luego, iterativamente se va reasignando las observaciones a los conglomerados hasta que algÃºn criterio de parada (funciÃ³n a optimizar) se satisface, o, se cumpliÃ³ el nÃºmero mÃ¡ximo fijado de iteraciones.

MÃ©todo K-means

Se recomienda normalizar las variables antes de implementar este algoritmo.

  • Centroide es un representate del cluster al que pertenece.

  • MetodologÃ­a:

    1. Asigna aleatoriamente un nÃºmero, de 1 a \(K\) , a cada una de las observaciones.
    2. Iterar hasta que la asignaciÃ³n de los cluster deje de cambiar
      1. Para cada uno de los \(K\) cluster, calcular el centroide. El k-Ã©simo centroide es el vector con las \(p\) medias de las variables para las observaciones en el k-Ã©simo cluster.
      2. Asignar cada observaciÃ³n al cluster donde el entroide estÃ© mÃ¡s cerca (donde cercanÃ­a se encuentra definida por la distancia Euclidiana).

  • Al inicio, el centroide es una observaciÃ³n; para las siguientes iteraciones, most likely ya no.

  • El algoritmo anterior garantiza que el valor del objetivo asociado a este modelo decrezca en cada etapa.

  • Este algoritmo es computacionalmente rÃ¡pido; usualmente se fija 20 o 25 como el mÃ¡ximo de iteraciones.

  • Cambiando la mÃ©trica usada para la funciÃ³n objetivo, se pueden obtener diferentes clusters.

  • Desventajas:

    • Usualmente no se encuentra un Ã³ptimo global, solo Ã³ptimo local; pero esto puede mitigarse permitiendo un mayor nÃºmero de iteraciones del loop del algoritmo.
    • Es sensible a outliers, debido al uso de medias; pero se puede mitigar vÃ­a el uso de mediodes (usar mediana en vez de media), empleando la mÃ©trica \(L_1\) en vez de \(L_2\) tambiÃ©n.

Silueta

  • Representa quÃ© tan bien estÃ¡ agrupada la observaciÃ³n, en su conglomerado respectivo.

  • Mayor silueta, mayor similitud de la observaciÃ³n a las observaciones en su conglomerado.

  • EstÃ¡ entre \(-1\) y \(1\), pero un valor negativo indica que la observaciÃ³n no pertenece a un cluster adecuado.

  • El ancho de silueta de la i-Ã©sima observaciÃ³n es definida por \(\text{ sil }_i = \dfrac{b_i - a_i}{max \left( a_i, b_i \right)}\) .

  • Definiciones:

    • \(a_i\) : Distancia promedio entre la observaciÃ³n \(i\) y todas las otras observaciones en el mismo conglomerado que i.
    • \(b_i\): Denota la mÃ­nima distancia promedio de \(i\) a las observaciones que estÃ¡n en otros conglomerados.

  • Silueta con valor 1 indica que esa observaciÃ³n estÃ¡ perfectamente asignada en su cluster respectivo.

  • La medida de silueta se define como \(\bar{s} = \dfrac{1}{n} \displaystyle{ \sum_{1}^{n} \text{ sil}_i }\)

Criterio de Calinski-Harabasz

  • Valor por calcular:

\[ \dfrac{(n-k) \times \text{ Suma de cuadrados entre los grupos }}{(k-1) \times \text{ Suma de cuadrados dentro de los grupos }} \]

  • Mayor valor, mejor agrupaciÃ³n de los objetos en conglomerados.

Ambos criterios, silueta y Calinski-Harabasz, sirven para cualquier tipo de conglomeraciÃ³n, no solo para K-means.
